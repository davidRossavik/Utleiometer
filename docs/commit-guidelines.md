# Commit messages 


## Why having a commit template? 
Using a commit template establishes a shared understanding for writing the commit messages, which improves consistency and makes the repository easier to maintain and understand. 


## Format
    type(scope*): what and why has been done
    Co-authored by: name of contributor**

*scope is optional \
**only co-author if pair-programming

### Example
    feat(ui): add search bar to home page
    Co-authored: username


## Type

- **feat** → new feature  
- **fix** → bug fix  
- **docs** → documentation only  
- **style** → formatting, no logic change  
- **refactor** → code restructuring without behavior change  
- **test** → add or modify tests  
- **chore** → maintenance, configs, dependencies  
- **perf** → performance improvements  


## Scope

- **components** → user interface logic
- **utils** → backend
- **data** → changing data storage structure/logic
